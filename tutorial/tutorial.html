Este é o guia oficial de como usar o software Sendex. Selecione as opções para ver a explicação:


Enviar E-mail

Para enviar e-mail basta preencher os campos Para, Assunto e Conteúdo, selecionar arquivo é opcional. Por fim, clicar em Enviar. Após enviar, o programa deve retornar uma mensagem de que foi enviado com sucesso, e então não necessita enviar novamente. Mas se retornar uma das seguintes mensagens de erro:

  • Preencha todos os campos: isso significa que você não preencheu os campos Para e/ou Assunto.
  • Não enviado: pode ser que o e-mail informado no campo "Para" não exista, verifique o que você digitou. Se estiver correto as causas para esse problema pode ser ausência de servidor local SMTP, dados de acesso ao servidor SMTP estão incompatíveis com o programa, e outros. Verifique as instruções nas instruções sobre Autenticação.



    • Autenticação

    Existem três dados que são referentes à autenticação no Sendex: e-mail, senha e registro:

  • e-mail: endereço de e-mail registrado no seu servidor SMTP.
  • senha: senha registrada no seu servidor SMTP (senha da conta de e-mail acima).
  • registro: valor qualquer. Serve de senha para entrar no programa.

    • Quando o programa é instalado você precisa atualizar a Autenticação, porque esses três dados apresentados vêm configurados como sendex, esperando que você atualize para os dados corretos. Mas se você alterou a senha da conta de email e já alterou no servidor SMTP, também é necessário atualizar aqui a autenticação, ou se alterar de conta de email, entre outras. Para atualizar basta seguir até a tela de Autenticação e preencher conforme abaixo:

    Atualizando pela 1ª vez
  • Conta do atual G-mail: digite sendex, essa é a configuração padrão inicial.
  • Senha do atual e-mail: digite sendex.
  • Registro atual: digite sendex.
    • Atualizar para
  • Conta do novo G-mail: conta de e-mail registrada no servidor SMTP.
  • Senha do novo e-mail: senha registrada no servidor SMTP (senha da conta).
  • Novo registro: informe qualquer valor (NÃO PODE SER ESQUECIDO) e use ele para entre no programa.

    • Não é a 1ª vez

    Dos dados atuais, informe o que está sendo utilizado até o momento. Dos novos dados, faça como foi feito na primeira vez (veja acima).


    Se você não se sentir confortável em informar a conta e a senha da mesma, poderá criar uma conta de e-mail apenas para isso. Mas lembre-se, deve ser a mesma registrada no servidor SMTP.

    Caso ainda não tenha seu agente SMTP devidamente instalado e configurado, siga o artigo: instalar e configurar um agente SMTP.


    Solicitar Logs

    Os logs são registro das atividades dentro do sistema, contém as informações de hora e ação, exemplo: email enviado com sucesso - [24/07/2021 às 14:19:33], sem envolver os dados de e-mails remetentes e do servidor. Porém, todas as atividades são registradas para que por algum motivo seja possível realizar análise do que foi feito.

    Ao clicar no botão de solicitação de logs, os registros serão enviados para o e-mail registrado no servidor SMTP, sem perguntar se deseja enviar para outra conta. A solicitação deve ser bem sucedida retornando uma mensagem positiva do envio. Caso retorne um erro, verifique as instruções sobre Autenticação.
